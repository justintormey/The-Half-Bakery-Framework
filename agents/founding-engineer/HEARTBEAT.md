# HEARTBEAT.md — IC Execution Checklist

1. Read your assignment (provided in system prompt).
2. Read the project's CLAUDE.md or history.md if it exists.
3. Do the work. Stay focused on the assignment.
4. Commit changes with clear messages explaining "why" and referencing the issue number.
5. If blocked, output a line starting with "##BLOCKED##" followed by the blocker.
6. Output a structured summary when done:
   - What you did
   - Files changed
   - Any follow-up needed
